# ADR-0009: ペインジャンプ後の自動終了をオプション化

## Status

Accepted

## Date

2026-03-20

## Context

baton の TUI でセッションを選択してペインジャンプすると、`tea.Quit` により baton が自動終了する設計だった。

tmux ポップアップ（`display-popup`）で baton を起動する運用が定着し、ジャンプ後にポップアップが閉じるのは都合が良い場面もあるが、複数セッションを連続的に切り替えたい場合は毎回ポップアップを開き直す必要があり不便だった。

また、通常のペイン内で baton を常駐させるケースでは、ジャンプ後も TUI に戻ってセッション一覧を確認し続けたいという要件があった。

## Decision

デフォルトの挙動を「ジャンプ後も TUI を維持」に変更し、従来の自動終了動作は `--exit` フラグで明示的に選択する方式にする。

変更箇所:

1. `main.go`: `--exit` フラグ（`bool`, default `false`）を追加し、`tui.NewModel()` に `exitOnJump` として渡す
2. `Model` 構造体: `exitOnJump bool` フィールドを追加
3. `Update()` の `JumpDoneMsg` 処理: `exitOnJump` が `true` なら `tea.Quit`、`false` なら `jumping = false` で TUI に戻る
4. `updateSubMenu()` のサブメニュー Enter 処理: 同様に `exitOnJump` で分岐

## Rationale

- **デフォルトは継続**: 複数セッションの連続操作が主要ユースケースであり、「戻れる」方が便利な場面が多い
- **`--exit` で従来動作を維持**: tmux ポップアップ起動スクリプトなど、ジャンプ後に閉じたい場合は `baton --exit` で対応
- **破壊的変更を受け入れる**: デフォルト挙動の反転だが、baton はまだ初期段階であり利用者も限定的。tmux popup スクリプトに `--exit` を追加するだけで移行完了

比較した代替案:

- **`--stay` フラグ（デフォルト終了を維持）**: 従来互換性は保たれるが、主要ユースケース（連続操作）で毎回フラグ指定が必要になり不便
- **config.yaml で設定**: フラグ1つのためにファイル設定を導入するのは過剰。将来的に設定項目が増えた場合に検討

## Consequences

### Positive

- 複数セッションの連続的な切り替えが、TUI を再起動せずに行える
- tmux ポップアップ利用時は `--exit` を指定するだけで従来と同じ挙動
- 非同期パス（`JumpDoneMsg`）と同期パス（サブメニュー）の両方で一貫した分岐ロジック

### Negative

- 既存のスクリプト・エイリアスがジャンプ後の自動終了を前提としている場合、`--exit` の追加が必要（破壊的変更）
- サブメニュー経由のジャンプは同期実行のため、通常パス（非同期 `JumpDoneMsg`）との実装の非対称性が残る（コメントで明示済み）
