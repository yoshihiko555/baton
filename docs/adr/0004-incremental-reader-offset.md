# ADR-0004: IncrementalReader による offset 管理

## Status

Accepted

## Date

2026-03-02

## Context

Claude Code のセッションログは JSONL 形式で追記される。ファイルが更新されるたびに全体を再パースするのは非効率であり、大量のセッションを監視する場合にパフォーマンス問題が発生する。

また、ログファイルはローテーション（切り詰め・再作成）される可能性がある。

## Decision

`IncrementalReader` を実装し、offset ベースの差分読み取りとローテーション検出を行う。

- ファイルの現在サイズを記録し、前回読み取り位置から差分のみをパース
- ファイルサイズが前回より縮小した場合はローテーションとみなし、先頭から再読み取り

## Rationale

- 差分読み取りにより、大量セッション監視時のパフォーマンスを確保
- ローテーション検出により、ログ管理による不整合を防止
- `tail -f` に相当する動作を Go で実現

## Consequences

### Positive

- ファイル更新時の処理量が最小限（差分のみ）
- ローテーション時も自動で状態がリセットされる

### Negative

- offset 管理の実装が必要で、バグが入り込みやすい（テストで補完）
- ファイルの中間部分が書き換えられるケースには対応しない（追記専用を前提）
