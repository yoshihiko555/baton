# ADR-0012: Attention セクションと安定順セッション一覧

## Status

Accepted

## Date

2026-04-19

## Context

従来の TUI 左ペインは `WAITING / ERROR / WORKING / IDLE` の状態グループで都度並び替えていた。この方式は監視用途では見やすい一方、状態変化が頻繁なセッションを上下移動で選択している最中に一覧の物理位置が変わり、意図した次のセッションへ移動しづらい問題があった。

特に以下の条件で操作性が悪化していた。

- `Thinking` / `ToolUse` / `Waiting` の遷移が短い間隔で発生する
- 同一プロジェクト内に複数セッションがあり、連続で比較・切り替えしたい
- 「今見えている一覧の 1 つ下/上」を選びたいが、再ソートで相対位置が崩れる

一方で、`Waiting` セッションは引き続き目立たせる必要がある。

## Decision

左ペインを以下の 2 層構成に変更する。

1. **Attention セクション**
   - `Waiting / Working / Idle` の件数を上部に表示する
   - 実セッションとして表示するのは `Waiting` のみとする
   - 最大 5 行まで表示する
   - `w` キーで `Waiting` セッションを巡回する（ショートカット表示は下部アクションバーに置く）
2. **メイン一覧**
   - 状態ではなく `project -> tool -> PID` の安定順で表示する
   - プロジェクト単位のヘッダーを持つ
   - 各セッションの状態は並び順ではなくバッジ・色で表現する

## Rationale

- **操作の基準と状態強調を分離できる**: 一覧の座標系は固定しつつ、要対応セッションだけを別枠で強調できる
- **相対移動が安定する**: 状態変化で行が上下してもメイン一覧の並びが変わらないため、連続した上下移動の意図を維持しやすい
- **監視用途も維持できる**: `Waiting` は Attention から即座に把握でき、`Working` / `Idle` の全体感も同じ場所で確認できる

比較した代替案:

- **状態グループのまま一時凍結**: 実装は軽いが、凍結タイミングと解除条件の理解が必要で挙動が不透明になりやすい
- **Monitor/Browse のモード分離**: 柔軟だが、TUI の認知負荷が増える
- **件数だけの Attention**: コンパクトだが「どのセッションが要対応か」を別途探す必要がある

## Consequences

### Positive

- 一覧の上下移動が安定し、複数セッションの比較がしやすくなる
- `Waiting` の視認性を維持しつつ、`Working` / `Idle` の分布も上部で把握できる
- フィルタ適用時も同じ並び方ルールを再利用できる

### Negative

- 従来の状態別グループ表示に慣れている場合、`Idle` / `Working` の塊感は弱くなる
- `Attention` 行数が増える分、低い画面高ではメイン一覧の表示行数が減る
